# 시나리오 — 이 프로젝트의 원천 진실

devspace의 모든 기능은 이 디렉토리의 **자연어 사용자 시나리오**로 기술된다.
리뷰는 이 문서들에 대해서만 이루어지고, 코드는 시나리오를 통과시키기 위한
수단이다.

## 규약

- 시나리오 하나 = 마크다운 파일 하나 = E2E 테스트 파일 하나(`e2e/*.spec.ts`).
  테스트의 각 `test.step` 제목은 시나리오의 번호 매긴 단계와 1:1로 대응한다.
- 시나리오는 **전제 / 흐름 / 정리** 세 부분으로 쓴다. 흐름의 각 단계는
  "사용자가 무엇을 하면, 무엇이 보인다"의 형태로, 구현 용어 없이 쓴다.
- E2E 테스트는 실제 브라우저, 실제 devcontainer 샌드박스, 실제 GitHub
  테스트 레포를 통과한다. 목(mock) 금지.

## 종류

| 위치 | 역할 |
| --- | --- |
| `golden-path.md` | 제품의 존재 이유인 핵심 흐름. 이것이 통과할 때까지가 1차 목표. |
| `regressions/` | 개발 도중 발견된 회귀성 이슈를 고정하는 시나리오. 항상 통과해야 한다. |

## 진행 방식

1. 골든패스 시나리오 테스트는 처음에는 **실패한다**. 이것이 목표 지점이다.
2. 기능을 쌓아 골든패스를 단계별로 통과시킨다.
3. 도중에 발견된 버그는 먼저 `regressions/`에 시나리오로 고정하고
   (실패하는 테스트), 그 다음 고친다.
