# 회귀: 종료 상태에서 입력이 조용히 사라지지 않는다

종료/실패 상태에서 사용자의 동작이 조용히 증발하던 버그(이슈 C, CLAUDE.md
"조용한 실패 금지" 위반)를 고정한다. PR이 열린(`pr-opened`) 세션의 입력창이
계속 활성이라, 여기서 보낸 메시지는 무효 전이로 예외가 되고 상위 `.catch`가
`console.error`로만 삼켜 **UI엔 아무 반응 없이 사라졌다.** `PR 만들기` 버튼도
클릭 후 비활성화되지 않아 두 번 누르면 같은 방식으로 조용히 실패했다.

## 전제

- 골든패스처럼 devspace 서버가 떠 있고, `E2E_REPO`·토큰·Codex 인증이 설정되어
  있다.
- 골든패스를 끝까지 진행해 PR이 열린 상태(`pr-opened`)까지 도달한다.

## 흐름

1. PR이 열린 뒤, 입력창은 **비활성**이다 — 종료된 세션에 메시지를 보내
   조용히 사라지게 만들 수 없다(활성이라면 보낸 메시지가 명확한 안내와 함께
   처리되어야 한다).
2. `PR 만들기` 버튼도 클릭 이후 **비활성**이라, 다시 눌러 중복 PR 시도가
   조용히 실패하는 일이 없다.

## 정리

- 테스트가 만든 PR은 닫고, PR 브랜치는 삭제한다. 세션(샌드박스)은 종료한다.
