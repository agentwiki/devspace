# 골든패스: 채팅으로 코드 수정을 시키고 PR을 받는다

devspace가 존재하는 이유. 이 시나리오가 처음부터 끝까지 통과하면
"나 혼자 쓰는 Claude Code 웹"의 최소 형태가 완성된 것이다.

## 전제

- devspace 서버가 내 서버에 떠 있다.
- 작업 대상으로 쓸 **테스트용 GitHub 레포**(`E2E_REPO`)가 지정되어 있고,
  devspace에는 그 레포를 클론하고 PR을 만들 수 있는 토큰이 설정되어 있다.
- Codex 구독 계정 인증(`auth.json`)이 설정되어 있다.

## 흐름

1. 브라우저로 devspace 주소에 접속하면 채팅 화면이 보인다.
2. 작업할 레포로 `E2E_REPO`를 지정하고 새 세션을 시작한다.
3. "샌드박스 준비 중"이라는 상태가 보이고, 이윽고 레포가 클론된
   샌드박스가 준비되었다는 메시지가 채팅에 나타난다.
4. 채팅 입력창에 다음과 같이 보낸다:
   > README.md 맨 아래에 "Hello from devspace" 한 줄을 추가해줘.
5. 에이전트가 작업하는 진행 상황이 채팅에 순차적으로 나타난다.
6. 작업이 끝나면 무엇이 바뀌었는지 요약(변경된 파일과 diff)과 함께
   **PR 만들기** 버튼이 나타난다.
7. **PR 만들기**를 누르면, 채팅에 생성된 PR 링크가 나타난다.
8. 그 링크의 PR은 실제로 `E2E_REPO`에 열려 있으며, diff에는 README.md에
   "Hello from devspace" 한 줄이 추가된 내용이 들어 있다.

## 정리

- 테스트가 만든 PR은 닫고, PR 브랜치는 삭제한다.
- 테스트가 만든 세션(샌드박스)은 종료한다.
